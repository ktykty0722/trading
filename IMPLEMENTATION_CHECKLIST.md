# 자동매매 구현 체크리스트 & To-Do (실전 운영용)

이 문서는 기존 리포지토리의 자동매매 구조를 기준으로, **실전에서 바로 적용 가능한 구현 체크리스트**와 **우선순위 To-Do**를 정리한 문서다.

---

## 0) 먼저 결정할 것 (시장/시간대 단일화)

- [ ] **거래 시장을 단일 선택**: `KRX` 또는 `US` (혼합 운영 금지)
- [ ] **기준 시간대 단일화**
  - KRX: `Asia/Seoul`
  - US: `America/New_York` (로그/UI에서만 KST 병기)
- [ ] 거래시장별 변동성 게이트 분리
  - KRX 전략: `VKOSPI` 사용
  - US 전략: `VIX` 사용
- [ ] 장 운영 시간 상수화 (scheduler/guard/monitor에서 공통 사용)

---

## 1) 아키텍처 체크리스트 (필수)

### 1-1. 데이터/피처 파이프라인
- [ ] 전일 OHLCV + 최소 60~120 거래일 히스토리 수집
- [ ] 이벤트 데이터 수집: 실적, 공시, 거래정지/관리종목, 배당/권리락
- [ ] 기술지표 계산: ATR, RSI, ADX, 추세/모멘텀 계열
- [ ] 유동성 지표 계산: 20일 평균 거래대금, 스프레드/체결 가능성
- [ ] 외부 API 호출 `timeout(connect, read)` 강제 및 재시도 정책 통일

### 1-2. 점수화/신호 생성
- [ ] 복합 점수(composite score) 산출식을 코드/문서에 명시
- [ ] 점수 임계치 하드코딩 금지 (`0.3` 고정 금지)
- [ ] 임계치는 Walk-forward + OOS 결과로만 확정
- [ ] LLM은 **감성/요약 단계 전용**으로 제한 (최종 주문 판단 금지)

### 1-3. 주문/체결 상태머신
- [ ] 상태 전이 표준화: `buy_ordered → holding → sell_ordered → sold`
- [ ] 부분체결/미체결/취소/거부 상태를 별도 코드로 저장
- [ ] 장 마감 전 미체결 정리 정책 정의 (정정/취소/다음날 이월 금지)
- [ ] 재기동 시 reconcile 루틴으로 주문/포지션 정합성 복구

## 2) 리스크 프레임워크 체크리스트 (핵심)

### 2-1. 매수 전 글로벌 게이트
- [ ] 변동성 게이트
  - KRX: `VKOSPI > 30` 신규매수 중단 (초기값)
  - US: `VIX > 25~28` 신규매수 중단 (초기값)
- [ ] 손실 게이트: 일손실 > 3%면 신규매수 중단
- [ ] 누적 MDD 게이트
  - MDD ≤ -5%: 포지션 크기 50% 축소
  - MDD ≤ -10%: 신규매수 전면 중단 + 수동 리뷰

### 2-2. 종목/포트폴리오 게이트
- [ ] 보유 종목 수 상한: 최대 5
- [ ] 동일 섹터 상한: 최대 2 (집중도 제한)
- [ ] 기존 보유와 상관계수 > 0.7 종목 진입 금지
- [ ] 이벤트 블랙아웃: 실적발표 ±2일 진입 금지
- [ ] 유동성 필터
  - 20일 평균 거래대금 하한
  - 최소 주가 하한(페니스톡 제외)
  - 거래정지/관리종목/상하한가 근접 종목 제외

### 2-3. 포지션 사이징/청산
- [ ] ATR 기반 리스크 사이징 (계좌 리스크 1%/종목)
- [ ] 진입: 지정가 원칙 (시장가 금지)
- [ ] 청산 규칙을 ATR 기반으로 통일
  - TP: +2.0 ATR
  - SL: -1.0 ATR
  - Trailing: 최고가 대비 -1.5 ATR
- [ ] 시간 청산: 5~10 영업일 범위에서 백테스트로 확정

## 3) 운영 스케줄 체크리스트 (예시)

> 아래는 US 기준 ET 예시. KRX는 동일 논리를 KST 장시간으로 치환.

- [ ] 08:00~09:20: 데이터 수집/피처 계산/점수화
- [ ] 09:20~09:29: 이벤트/유동성/리스크 게이트
- [ ] 10:30~10:35: 진입 실행 (지정가, 분할진입 허용)
- [ ] 10:35~15:50: 1분 주기 청산 모니터링
- [ ] 15:50~16:00: 미체결/익일 이벤트 정리
- [ ] 17:00: 일일 리포트 생성/전송

## 4) 백테스트/검증 체크리스트 (배포 전 필수)

- [ ] 수수료/슬리피지 반영 (편도 0.10~0.15% 초기 가정)
- [ ] 체결 불가 조건 반영 (거래정지, 상하한가 근접, 유동성 부족)
- [ ] 파라미터 스윕
  - 점수 임계치: 0.2/0.3/0.4/0.5
  - TP/SL/Trailing ATR 배수
  - 보유기간 제한
- [ ] Walk-forward validation
- [ ] OOS 성능 기준 미달 시 배포 차단
- [ ] 레짐별 성능 분해 (상승/횡보/급락장)

---

## 5) ML 운영(MLOps) 체크리스트

- [ ] 재학습 주기: **월 1회 기본** (주간 자동 재학습 금지)
- [ ] 베이스라인 모델 우선: LightGBM/XGBoost
- [ ] 자동 배포 금지: 승격 게이트(Sharpe, MDD, WinRate, Turnover) 통과 시만 배포
- [ ] 드리프트 모니터링
  - PSI > 0.25 경보
  - 성능 급락 시 롤백 후보 자동 지정
- [ ] 모델/피처/파라미터 버전 태깅 및 아티팩트 보관

---

## 6) 안정성/보안/관측성 체크리스트

- [ ] 환경변수 startup validation (API key/계좌/DB 누락 즉시 실패)
- [ ] CORS 운영값 제한 (`*` 와 credentials 조합 금지)
- [ ] 구조화 로깅 통일 (print 금지)
- [ ] 장애 알림 (주문 실패율, 지연, 데이터 수집 실패)
- [ ] 핵심 메트릭 대시보드
  - fill rate, 슬리피지, 전략 손익, MDD, 섹터 노출

## 우선순위 To-Do List (실행 순서)

- [ ] 시장/시간대 단일화 + 거래시간 상수 정비
- [ ] VIX/VKOSPI 게이트를 시장별로 분리
- [ ] LLM을 최종 주문 의사결정에서 제거 (감성 보조로만)
- [ ] ATR 기반 TP/SL/Trailing으로 청산 규칙 통일
- [ ] 이벤트 블랙아웃 + 유동성 필터 추가

- [ ] 섹터 상한 2종목 + 상관계수 필터(>0.7 제외) 적용

- [ ] 백테스트에 슬리피지/수수료/체결불가 반영
- [ ] 점수 임계치/보유기간/ATR 배수 Walk-forward 튜닝

### P2 (운영 고도화)

- [ ] 운영 대시보드/리포트 자동화 고도화

## 수용 기준(Definition of Done)

- [ ] 종목 진입 전 모든 게이트가 로그로 검증 가능
- [ ] 주문/체결 상태 전이가 재기동 후에도 복구됨
- [ ] 백테스트(OOS 포함) 결과가 실거래 가정(수수료/슬리피지) 포함
- [ ] 모델 변경은 승격 기준 통과 이력이 없으면 배포되지 않음
- [ ] 일일 리포트에 손익 + 리스크 지표 + 규칙 위반 내역 포함
